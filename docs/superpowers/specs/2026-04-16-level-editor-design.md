# 关卡编辑器设计

## 概述

独立网页编辑器 `editor.html`，供开发者在 800×600 可视化画布上编辑关卡数据，导出为 JSON 后复制到 `src/levels/` 目录下使用。

## 技术选型

- 纯 HTML + CSS + JavaScript，无框架，无构建步骤
- Phaser 3 用于画布渲染和节点/边交互
- 同游戏共用 Phaser CDN，可直接复用现有的绘制逻辑

## 数据模型

内存中的关卡对象（与现有 `level*.js` 格式一致）：

```json
{
  "id": 1,
  "name": "",
  "difficulty": "简单",
  "startNode": 0,
  "homeNode": 7,
  "initialEnergy": 120,
  "nodes": [
    { "id": 0, "x": 400, "y": 545, "isStart": true, "isHome": false, "isDead": false, "label": "起点" }
  ],
  "edges": [[0, 1], [1, 2]],
  "food": [{ "nodeId": 2, "type": "leaf" }],
  "wings": [{ "nodeId": 3 }],
  "fastTravel": [[2, 6]],
  "npcs": [
    { "id": "frog_1", "type": "frog", "edgeA": 3, "edgeB": 5, "sneakEnergyCost": 20 }
  ]
}
```

## 布局

```
┌──────────────────────────────────────────────────────────┐
│  [工具条]  导出JSON  导入JSON                              │
├─────────────────────────┬───────────────────────────────┤
│                         │  [标签页]                       │
│                         │  节点 │ 边 │ 道具 │ NPC │ 基础  │
│    800×600 Phaser 画布   │  ─────────────────────────── │
│    （可拖拽节点）         │  [面板内容]                   │
│                         │                               │
└─────────────────────────┴───────────────────────────────┘
```

左侧：Phaser 渲染区域
右侧：Tab 切换的属性面板（约 320px 宽）

## 工具条模式

| 模式 | 图标 | 点击画布行为 |
|------|------|-------------|
| 选择 | 箭头 | 选中节点高亮；拖拽节点移动位置 |
| 添加节点 | 圆圈+ | 点击空白处添加节点，自动分配最小可用 id |
| 添加边 | 连线 | 依次点击两个节点 → 创建边 |
| 删除 | 垃圾桶 | 点击节点或边 → 删除（节点自动删除关联边）|
| 放置食物 | 苹果 | 点击节点 → 在该节点添加/移除食物（循环 leaf/berry/apple）|
| 放置翅膀 | 翅膀 | 点击节点 → 切换翅膀放置状态 |
| 添加快速通道 | 闪电 | 依次点击两个节点 → 创建快速通道对 |
| 添加 NPC | 青蛙 | 依次点击两个节点 → 在该边上放置青蛙，弹出 NPC 配置浮窗 |

选中节点时，画布下方显示节点坐标。

## 右侧面板标签页

### 节点标签页
表格列出所有节点，选中行高亮。属性区显示：
- id（只读）、x / y（数字输入）、label（文字输入）
- isStart / isHome / isDead 复选框（互斥：勾选 Start 或 Home 自动取消其他节点的同名标记）

### 边标签页
列表显示所有边，格式为 `3 ↔ 5`，点击行删除该边。

### 道具标签页
分三个小节：
- **食物**：每行显示 `节点N: leaf|berry|apple`，可切换类型
- **翅膀**：每行显示节点 id，可删除
- **快速通道**：每行显示 `[A ↔ B]`，可删除

### NPC 标签页
列表显示所有 NPC，点击展开详细配置浮窗：
- id（文本输入）
- type（只读，当前仅支持 frog）
- edgeA / edgeB（节点下拉选择）
- sneakEnergyCost（数字输入，默认 20）
- sleepDuration（滑块 1000~10000ms，默认 5000）

### 基础信息标签页
- name（文本输入）
- difficulty（select: 简单 / 普通 / 困难）
- initialEnergy（数字输入，默认 120）
- startNode / homeNode（节点下拉选择）

## 导出 / 导入

- **导出 JSON**：点击按钮 → 生成格式化 JSON → 复制到剪贴板 → 显示成功提示
- **导入 JSON**：点击按钮 → 弹出文本框 → 粘贴 JSON → 解析并加载到编辑器；解析失败显示错误提示

## 初始状态

编辑器打开时加载默认空白关卡（空节点数组），用户从零开始构建。

## 验收标准

1. 能在画布上添加、移动、删除节点
2. 能通过节点对创建边，能删除边
3. 节点属性（起/终点/死路标签）在面板和画布上同步显示
4. 食物/翅膀/快速通道/NPC 的添加和删除正确反映在导出 JSON 中
5. 导出 JSON 格式可直接粘贴进 `src/levels/level{N}.js` 的 `export const LEVEL{N} = { ... }` 中使用
6. 导入已有 JSON 能正确还原关卡布局
