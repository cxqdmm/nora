# 青蛙障碍系统设计

**日期**: 2026-04-14
**状态**: 设计完成，等待实现

---

## 概述

在现有关卡中引入青蛙（NPC 障碍），玩家可以使用道具（小刀 / 催眠药）解决，或选择偷偷溜过但会消耗能量。道具通过吃食物随机掉落。首个实验关卡为第一关。

---

## 核心玩法

### 青蛙位置

青蛙占据一节树枝的中间位置（不在节点上）。当毛毛虫进入该树枝的任意一个端点节点时，青蛙激活，进入攻击状态，暂停毛毛虫移动。

```
节点 A ——[青蛙坐在树枝中间]—— 节点 B
毛毛虫到达 A → 青蛙激活
毛毛虫到达 B → 青蛙激活
```

### 青蛙状态机

| 状态 | 描述 |
|------|------|
| `idle` | 青蛙在树枝中间待命，树枝正常可通行 |
| `active` | 毛毛虫触发，青蛙扑出，暂停游戏，弹出道具面板 |
| `sleeping` | 被催眠药击中，睡觉中，玩家可在倒计时内通过 |
| `dead` | 被小刀杀死，青蛙消失，树枝永久畅通 |

**催眠药时长**: 5 秒（硬编码，后续可 NPC 个性化）

### 解决方式

| 方式 | 效果 |
|------|------|
| **小刀** | 飞刀动画从毛毛虫飞向青蛙 → 青蛙永久消失，树枝畅通 |
| **催眠药** | 青蛙睡着 5 秒 → 倒计时内可通过该树枝 → 倒计时结束返回 `active` 状态 |
| **偷偷溜** | 扣固定能量值 → 强制通过该树枝到达目标节点 → 青蛙仍存在 |

## 道具系统

### 获取方式

吃食物时有概率掉落道具。掉落概率在 `CONFIG.ITEMS.DROP_CHANCE` 中配置。

| 食物类型 | 可掉落道具 |
|----------|-----------|
| `leaf` | 小刀（20% 概率） |
| `berry` | 催眠药（20% 概率） |
| `apple` | 小刀（15%）/ 催眠药（15%） |

道具无限持有，不限数量。

### 道具类结构

```
ItemModule（新增，管理背包）
  ├── KnifeItem（永久消耗品）
  └── SleepPotionItem（永久消耗品）
```

使用后从背包中移除。

## NPC 系统（抽象基类）

为未来扩展其他 NPC 做准备，抽象基类设计：

```
NPCModule（基类）
  ├── _state: 'idle' | 'active' | 'sleeping' | 'dead'
  ├── _sleepDuration: number（毫秒，可被子类覆盖）
  ├── activate(): void
  ├── sleep(durationMs): void
  ├── kill(): void
  ├── isBlocking(): boolean
  ├── onPlayerPass(callback): void
  └── render(state): void

FrogNPC extends NPCModule（子类，定制行为）
```

子类可覆盖：
- `_sleepDuration` — 睡眠时长
- `activate()` — 激活时的自定义行为（动画、音效）
- `render()` — 不同状态的视觉呈现

## 交互流程

```
毛毛虫点击目标节点 → 开始移动
  ↓
到达触发节点（青蛙所在树枝的端点）
  ↓
青蛙 activate() → 状态变为 active
毛毛虫暂停，道具面板弹出
  ↓
玩家选择：
  ├── 点击小刀 → 飞刀动画 → 青蛙 kill() → 状态 dead → 继续移动
  ├── 点击催眠药 → 催眠动画 → 青蛙 sleep(5000ms)
  │     → 倒计时气泡显示 → 玩家通过 → 倒计时结束 → 状态 active
  └── 点击偷偷溜 → 能量 drain() → 强制移动到目标节点 → 青蛙仍 idle
```

## UI 设计

### 道具面板

- 屏幕中央弹出，半透明黑色背景遮罩
- 显示已拥有的道具图标（小刀 🗡️ / 催眠药 💤）
- 有道具时图标亮起；无道具时灰色不可点击
- 有道具时，图标周围有发光边框提示"可用"
- 面板底部显示"偷偷溜（消耗 ⚡ X 点能量）"按钮（始终可点）

### 催眠倒计时气泡

- 青蛙头上显示圆形倒计时气泡（5 → 0）
- 气泡随时间缩小并变红
- 到 0 时气泡消失，青蛙重新激活

### 飞刀动画

- 小刀从毛毛虫头部飞向青蛙
- 沿直线飞行，时长约 300ms
- 命中后青蛙消失，树枝恢复正常

## 第一关集成

在第一关中选一条分支树枝放置青蛙：

**选择**: 节点 3 → 节点 5 的树枝（左侧主干分叉）

```
节点 3（触发节点）——[🐸 青蛙]—— 节点 5
```

- 青蛙 `idle` 状态：玩家可以正常通过
- 玩家首次进入节点 3 或 5 时，青蛙激活
- 解决后树枝永久畅通
